REFERÊNCIA RÁPIDA - NAVEGAÇÃO COM FILTROS
========================================

✨ PROBLEMA RESOLVIDO:
Quando usuário navegava para detalhes e clicava em "voltar", perdia todos os filtros.
Agora mantém a URL anterior (com filtros, paginação, etc) preservada.

🔧 COMO FUNCIONA:

1️⃣ JavaScript detecta página de LISTAGEM
   ↓
   NavigationHistory.saveCurrentUrl()
   ↓
   sessionStorage['lastNavigationUrl'] = URL com filtros

2️⃣ Usuário navega para DETALHES/EDIÇÃO
   ↓
   URL anterior está salva no sessionStorage

3️⃣ Usuário clica em "Voltar"
   ↓
   NavigationHistory.goBack(fallbackRoute)
   ↓
   ✅ Vai para URL anterior COM FILTROS
   ❌ Se não existir, usa fallbackRoute


📌 EXEMPLOS DE USO:

Em qualquer template show/create/edit:

  <button onclick="window.NavigationHistory?.goBack('{{ route('termos.index') }}')" 
          class="btn btn-secondary">
      <i class="fas fa-arrow-left"></i> Voltar
  </button>

O 'route('termos.index')' é apenas FALLBACK se não houver URL anterior.


✅ PRÓXIMOS PASSOS:

1. Compilar assets:
   $ npm run build
   ou
   $ npm run dev (watch mode)

2. Testar em navegação real:
   - Listagem de Termos → Aplicar filtros → Abrir termo → Voltar
   - Deve manter filtros ✨

3. Verificar console (F12) para erros
